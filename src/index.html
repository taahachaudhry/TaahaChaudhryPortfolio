Portfolio Redesign

DESIGN

WEB

Web portfolio site application design

Github Repository: Site:

Objective

To redesign and develop my portolio site for a better, simpler, and more concise User Experience and User Interface for recruiters, developers, and users to be able to view my skills, work, and resume.

Design Process

I began this redesign by collecting feedback on my current portfolio site from other designers and recruiters. Two points were mainly derived from the feedback that I recieved. One is to remove unnecessary features and visual aesthetics (such as the hero section) to create a more concise and simpler UX. The second point is to understand what the content the users wants to see and restructuring the hierachy.

Based on the feedback that I gained from the user research that I conducted, I began writing out ideas and creating solutions to redesign the site. Thereafter, I began sketching low-to-mid fidelity wireframes and implementing the visuals using Sketch.

Solution

I first began by listing out the important components, which would dictate the structure of the application. The most important component is the display of my work, which is the purpose of this application, should be the first thing that users should be able to see, upon entering the site. Each project should also contain a process or walkthrough, or even a description, and anything else related to the projects. Other components included the about section, resume, and contact.

By understanding the components for the application and the feedback I recieved, I created ideas of how I was going to build the wireframes. I decided to remove unnecessary items, such as the hero components or quote, and remove animated functionality, and combine items where possible, such as displaying my skills through the project descriptions. I also wanted to create a smooter way to navigate through the site, so I decided to have a left-side navigation panel, which would also contain my contact information and social links. I also came up with a better hierarchy and structure for displaying project content.

Development

I decided to build this site myself using HTML, Sass, and JavaScript, along with some libraries, such as Animate.css, jQuery, and Masonry. I also used Gulp to build tasks, such as compiling my Sass files, linting, and browserify to use require in the browser by bundling up my dependencies. The biggest advantage I had using Gulp was a way to create a separate directory, which contained the HTML files, compiled Sass files, JavaScript files, and assets. By having this separate directory of compiled code, I was able to use Github pages to host my site, while still being able to code using Gulp and SCSS, and commit code as I normally would to a main repository.
